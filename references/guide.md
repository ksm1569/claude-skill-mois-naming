# 행정안전부 공공데이터 공통표준 명명 가이드 (범용)

> **목적**: 공공·SI 프로젝트와 감리사업에서 **DB 컬럼/테이블, Java 클래스/패키지, 파일/디렉터리, REST 엔드포인트, 파라미터** 등 모든 식별자 명명을 행정안전부 공공데이터 공통표준에 맞춰 일관되게 작성·검증하기 위한 참조 문서.
>
> **AI 친화 설계**: 본 문서는 13,000건이 넘는 표준 사전을 컨텍스트에 직접 싣지 않는다. 대신 **표준 자료의 출처·구조·생성규칙·자주 쓰이는 약어**만 제공하여, AI가 본 문서와 (선택적으로) 원본 CSV를 함께 활용해 새 식별자를 표준에 맞게 만들거나 검증할 수 있도록 한다.

## 2. 행안부 표준약어의 생성 패턴

행안부 표준약어는 **자음 압축식(consonant compression)** + **의미보존 어근** 방식으로 만든다.

### 2.1 기본 원리

| 패턴 | 예시 |
|---|---|
| 영문 단어의 모음 제거 | NAME → `NM`, NUMBER → `NO`, AMOUNT → `AMT`, COUNT → `CNT` |
| 어근 보존 + 모음 제거 | MANAGER → `MNGR`, MEMBER → `MBR`, REGISTER → `REG` / `RGTR` |
| 한국어 → 의미 영문 후 압축 | 회원가입 → MEMBER+JOIN → `MBR_JOIN`, 일자 → DATE → `YMD` |
| 합성어는 `_` 로 결합 | `MBR_JOIN_YMD` (회원가입일자), `SCHL_NM` (학교명) |
| 일부는 한국어 자음 압축 | 휴대폰 → `HP`, 이메일 → `EML` |

### 2.2 검증 방법

1. 후보 약어가 행안부 단어/용어 사전에 **존재하는지 검색**
2. 존재하면 그대로 사용
3. 없으면 §2.1 패턴으로 직접 생성하되, 동일 어근에 대해 프로젝트 내 일관성 유지

## 6. 변환 워크플로 (한글 명칭 → 표준 식별자)

### 6.1 수동 절차

1. **의미 토큰 분리**: 한글 명칭을 의미 단위로 자른다. 예) "회원가입 신청자 목록" → 회원가입 / 신청자 / 목록
2. **표준 사전 검색**: 각 토큰을 §1의 단어 사전(또는 용어 사전)에서 검색
3. **약어 결합**: 검색된 약어를 `_`(SNAKE) 또는 case 변환으로 결합
4. **목적별 case 변환**: §5에 따라 컬럼/클래스/파일 등 목적에 맞춰 변환
5. **충돌 검사**: 같은 스코프(테이블/패키지/폴더) 안에서 동명 식별자가 있으면 상위 컨텍스트(부모 메뉴/도메인)를 prefix로 추가

### 6.2 Python 스니펫 (CSV가 어디 있든 동작)

```python
import csv

CSV_PATH = '/path/to/공통표준용어.csv'   # 또는 단어/도메인

def search(keyword: str, top: int = 5):
    """주어진 한글 키워드를 표준 사전에서 부분 매칭."""
    out = []
    with open(CSV_PATH, encoding='utf-8-sig', newline='') as f:
        for row in csv.DictReader(f):
            name = row.get('공통표준용어명') or row.get('공통표준단어명')
            abbr = row.get('공통표준용어영문약어명') or row.get('공통표준단어영문약어명')
            if name and abbr and keyword in name:
                out.append((len(name), name, abbr))
    out.sort()           # 짧은 매칭이 곧 어근에 가까움
    return out[:top]

print(search('회원가입'))
# → [(5, '회원가입월', 'MBR_JOIN_MM'), (5, '회원가입일', 'MBR_JOIN_DAY'), ...]
```

가장 짧은 매칭 결과의 약어가 그 단어의 표준 root이다.

### 6.3 도메인(데이터타입) 파싱

표준용어 CSV의 `공통표준도메인명` 컬럼(예: `명V100`, `수N15,4`, `연월일시분초D`)에서 DB 데이터타입을 추출한다. search.py의 `parse_domain()` 함수를 사용하거나 아래 규칙을 적용:

| 도메인 코드 | DB 타입 | 예시 |
|---|---|---|
| `V` | VARCHAR | `명V100` → VARCHAR(100) |
| `C` | CHAR | `연월일C8` → CHAR(8) |
| `N` | NUMBER | `수N15,4` → NUMBER(15,4) |
| `D` | DATE | `연월일시분초D` → DATE (길이 없음) |

```bash
# search.py로 도메인 정보 확인 (DB타입 컬럼에 파싱 결과 표시)
python scripts/search.py "회원가입일자" --exact
```

### 6.4 권장 자동화 단계

1. 프로젝트 시작 시 **단어/용어/도메인 CSV 3종**을 한 폴더에 두기 (위치는 자유)
2. 본 가이드와 위 스니펫을 참고해 **어근 매핑 dict**를 자동 빌드 (또는 수작업으로 §4 표 갱신)
3. 식별자 생성 시 dict 우선 → 미스 시 CSV 직접 검색 → 그래도 없으면 §2.1 패턴으로 신규 약어 생성하고 매핑 dict에 등록 (일관성 보장)

## 7. 표준에 없는 단어를 만들 때 (생성 규칙)

### 7.1 우선순위

1. **표준단어**(15156439)에서 동일·유사 어근 검색
2. 없으면 **표준용어**(15156379)에서 합성어 패턴 참고
3. 그래도 없으면 §2.1 자음 압축 패턴으로 직접 생성

### 7.2 자체 생성 시 지침

- 영어 어휘가 더 자연스러우면 **영어 우선**: 지도→`MAP`, 카드→`CARD`, 홈→`HOME`
- 가능한 **3~5자**로 제한 (행안부 평균에 맞춤)
- 새로 만든 약어는 **프로젝트 자체 어근표**(예: `project_abbreviation_dict.md`)에 한 줄로 등록해 일관성 유지
- 같은 의미에 두 약어를 쓰지 말 것

### 7.3 자체 생성 약어 등록 양식 (프로젝트별로 따로 작성)

```markdown
| 한글 | 약어 | 출처 | 비고 |
|---|---|---|---|
| 학부모 | PRNT | 자체생성 (parent 자음압축) | 표준 미존재 |
| 돌봄 | CARE | 자체생성 (영어 우선) | 표준 미존재 |
| 카테고리 | CTGRY | 자체생성 (CATEGORY 자음압축) | 표준은 CLSF(분류) 사용 권장 |
```

## 8. 감리 관점의 검증 체크리스트

SI 프로젝트 감리/품질검토 시 본 표준 준수 여부를 확인하기 위한 체크리스트.

- [ ] DB 컬럼·테이블이 행안부 단어/용어 사전과 매칭되는가
- [ ] 동일 의미가 두 약어로 분리되어 쓰이지 않는가
- [ ] 표준에 없어 자체 생성한 약어가 별도 문서에 등록되어 있는가
- [ ] 도메인 자료(15156442)의 데이터 타입을 컬럼에 적용했는가
- [ ] 동일 식별자가 한 스코프에서 충돌하지 않는가 (`_`/케이스 변환 포함)
- [ ] 약어 길이가 무리하게 길거나(8자 초과) 짧지(2자 미만) 않은가
- [ ] 영문약어가 모두 대소문자·자음 압축 일관 규칙을 따르는가

## 10. 라이선스·갱신

- 행정안전부 공공데이터는 **공공누리 제1유형(출처표시)** 또는 그에 준하는 조건으로 자유 활용 가능 (각 데이터셋 페이지의 라이선스 표기 확인)
